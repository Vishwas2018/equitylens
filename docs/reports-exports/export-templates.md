# Export Templates

> Canonical specifications for every report EquityLens produces — CSV, PDF, and structured JSON. Templates are versioned, immutable once published, ATO-aligned where applicable, and always carry the standard financial disclaimer. The engine computes the numbers; the renderer is a pure function of `(scenario_result_id, template_version, locale)`.

---

## 1. Principles

1. **Reproducible**: an export is fully determined by `(scenario_result_id, template_version, locale)`. Same inputs always produce a byte-identical PDF (no timestamps in content body) and identical CSV.
2. **Versioned**: template versions are immutable. Reports record the template version that produced them; reruns under a new template version are explicit user actions.
3. **ATO-aligned**: financial CSVs map to ATO line items where a direct correspondence exists; labels mirror the wording used by the ATO and SRO where possible to reduce friction during tax preparation.
4. **Disclaimer-bound**: every artifact carries the standard disclaimer block. Renderers refuse to emit an artifact without it.
5. **Audit-trailed**: every export writes a row to `reports` and `audit_logs` capturing requester, scenario, template version, ruleset version, engine version, output hash, and delivery method.
6. **PII-minimal**: exports never include TFNs, full DOBs, or unmasked bank details — even when the user typed them — because exports leave the platform's trust boundary.

---

## 2. Template Registry

Templates live in `apps/web/server/reports/templates/` and are registered in the `report_templates` table:

```sql
CREATE TABLE equitylens.report_templates (
  id                uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  slug              text NOT NULL,                      -- e.g. 'portfolio-summary'
  version           text NOT NULL,                      -- semver, e.g. '1.4.0'
  format            text NOT NULL CHECK (format IN ('pdf','csv','json','xlsx')),
  schema_ref        text NOT NULL,                      -- path to JSON schema for inputs
  disclaimer_ref    text NOT NULL,                      -- key into equitylens.disclaimers
  state             text NOT NULL DEFAULT 'draft'
                    CHECK (state IN ('draft','published','retired')),
  published_at      timestamptz,
  published_by      uuid REFERENCES equitylens.users(id),
  retired_at        timestamptz,
  spec_hash         text NOT NULL,                      -- sha256 of rendered spec
  created_at        timestamptz NOT NULL DEFAULT now(),
  UNIQUE (slug, version)
);

CREATE INDEX ON equitylens.report_templates (slug, state) WHERE state = 'published';
```

A DB trigger locks rows once `state = 'published'`; subsequent updates raise an exception. Retiring a template is a separate state transition that does **not** delete or modify prior published rows.

## 3. Common Building Blocks

### 3.1 Disclaimer Footer

Every PDF page and every CSV header row references the disclaimer. The disclaimer text is loaded from `equitylens.disclaimers` (versioned, legal-approved) and recorded against the report.

```text
This report is generated by EquityLens Pty Ltd ("EquityLens"). It is
general information only, does not take into account your personal circumstances,
and does not constitute financial, taxation, legal, or investment advice. Figures
are estimates produced by a software model based on inputs provided by you and
the tax ruleset version stated below; they may differ from amounts ultimately
assessed by the Australian Taxation Office, State Revenue Office of Victoria, or
other authorities. You should consult a registered tax agent, financial adviser,
or solicitor before making decisions. EquityLens does not guarantee the accuracy,
completeness, or currency of any figure herein.

Engine version: {engine_version}     Ruleset version: {ruleset_version}
Generated:     {generated_at_iso}    Report ID:       {report_id}
Template:      {template_slug}@{template_version}
```

Renderers fail with `MissingDisclaimerError` if the placeholder block is absent from the layout.

### 3.2 Identification Header

```text
EquityLens — {template_human_name}
Prepared for: {tenant_display_name}
User:         {user_display_name} ({user_email_masked})
Date:         {generated_at_human}
Scenario:     {scenario_name} ({scenario_id})
```

`user_email_masked` shows only the first character and the domain (`j***@example.com`). Full email is never printed.

### 3.3 Money Formatting

* Stored as bigint cents in the engine; rendered as `$1,234,567` (no decimals) for headline figures and `$1,234,567.89` where pennies matter (CGT calculation lines, settlement statements).
* Negative values shown as `-$1,234` in CSVs and `($1,234)` in PDFs (accounting convention).
* Currency code always `AUD`; templates assert and refuse other codes.

### 3.4 Date Formatting

* Long form: `1 July 2026` (PDF body)
* Short form: `01/07/2026` (PDF tables)
* ISO: `2026-07-01` (CSV, JSON)
* Financial year: `FY2026` (= year ending 30 June 2026, ATO convention)

### 3.5 Locale & i18n

Initial release is `en-AU` only. Renderer signature includes a `locale` parameter for forward compatibility; non-AU locales raise `UnsupportedLocaleError`.

## 4. CSV Templates

CSVs are UTF-8 with BOM (so Excel opens them correctly), `\r\n` line endings, and RFC 4180-quoted fields. Numbers are unquoted; strings containing `,`, `"`, or newline are quoted with `""` escaping.

### 4.1 Header Metadata

Every CSV begins with `#`-prefixed metadata lines so Excel users can see provenance without losing parseability (most CSV parsers respect comment prefixes; we also publish a paired `.meta.json` for strict parsers).

```csv
# EquityLens Export
# template: cashflow-annual
# template_version: 1.2.0
# scenario_id: 01HZK9...
# scenario_name: Hold all, refi 2028
# ruleset_version: FY2026-VIC-3
# engine_version: 2.4.1
# generated_at: 2026-05-19T03:14:22.881Z
# report_id: 01HZKA...
# disclaimer: see https://equitylens.com.au/disclaimers/v3
fy,property_id,property_label,gross_rent,vacancy_loss,property_mgmt_fee,...
```

## 5. PDF Templates

PDFs are rendered server-side with `@react-pdf/renderer`. Layouts use the same OKLCH design tokens as the web app (see `/ui-ux/design-system.md`), shipped via a shared `@equitylens/design-tokens` package.

### 5.1 Shared Layout

```
┌─────────────────────────────────────────────────────────────────────┐
│ EquityLens logo                                       {template_human_name}│
│                                          Generated {generated_at}    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│ [Identification header]                                             │
│                                                                     │
│ [Page content]                                                      │
│                                                                     │
├─────────────────────────────────────────────────────────────────────┤
│ {disclaimer footer — required on every page}                         │
│ Page {n} of {N}                            Report ID: {report_id}    │
└─────────────────────────────────────────────────────────────────────┘
```

### 5.2 `portfolio-summary` (v1.3.0)

One page (or two for >8 properties).

**Page 1 — Snapshot**
* Header tile band: total portfolio value, total debt, net equity, gross yield, pre-tax cashflow, after-tax cashflow, weighted LVR.
* Property table: label, suburb (no full address), purchase year, current value, debt, equity, LVR, annual after-tax cashflow.
* 10-year equity forecast chart (line, single series); accompanied by a data table for screen-reader and PDF text-extraction parity.
* Disclaimer footer.

**Optional Page 2** — Property table continuation if >8 rows.

### 5.3 `property-detail` (v1.2.0)

Multi-page, sectioned:

1. **Property overview** — label, type, suburb, purchase price/date, current value (with valuation source + date), debt, LVR, ownership structure.
2. **Loan summary** — current balance, rate, interest type, IO end date, P&I switch date, remaining term.
3. **Rental** — current rent, vacancy assumption, growth assumption.
4. **30-year cashflow forecast** — line chart + companion table (years × {rent, deductions, pre-tax cashflow, tax impact, after-tax cashflow}).
5. **Tax position** — marginal rate, ruleset version, deductible expense breakdown, depreciation schedule reference.
6. **Equity forecast** — value, debt, equity over time.
7. **CGT at-disposal estimate** — pre-computed for a hypothetical disposal at the assumption horizon.
8. **Land tax (VIC)** — annual figures.
9. **Assumptions** — explicit list of every assumption the engine used.
10. **Disclaimer** — full disclaimer block.

Every numerical figure on a chart is also present in a table so the PDF is fully accessible and parseable.

### 5.4 `scenario-comparison` (v1.1.0)

Up to four scenarios side-by-side.

* Header row: scenario name + one-line description + ruleset version per scenario.
* Comparison table rows: total portfolio value (end of horizon), net equity (EOH), cumulative after-tax cashflow, cumulative CGT exposure, peak LVR, year of first positive cashflow.
* Side-by-side bar chart of selected metrics + companion table.
* Per-scenario assumptions appendix.

### 5.5 `hold-vs-sell` (v1.0.0)

Two-column page comparing:
* **Hold**: 10-year after-tax cashflow path, equity path, terminal equity at year 10.
* **Sell now**: net proceeds after CGT and selling costs, projected portfolio value if re-invested per user's nominated alternative (or shown as cash if no alternative).
* Decision aids: break-even year for hold, sensitivity to rental growth ±1%, sensitivity to capital growth ±1%.
* Heavy disclaimer pull-quote: this is decision support, not a recommendation.

### 5.6 PDF Determinism

* `react-pdf` is configured with `fontSubsetting: true` and embedded fonts; the same content always produces the same byte stream.
* The `generated_at` in the footer is the only timestamp; it is **excluded** from the report's `output_hash`. The hash covers content + disclaimer + identification (minus the timestamp).
* CI runs `pnpm test:pdf:golden` which renders the standard fixture set and compares against golden files; visual diffs require explicit approval.

## 7. XLSX Templates

XLSX is a convenience wrapper over CSV for the same templates that emit CSV. We use SheetJS server-side.

Rules:
* One worksheet per logical CSV table; the first worksheet `_meta` holds the metadata block (template, scenario, ruleset/engine version, disclaimer link, output hash).
* Numbers are written as numbers (not strings); dates as ISO date cells.
* No formulas — XLSX is a data export, not a model.
* No macros, no VBA. The renderer rejects any template that attempts to inject either.

---

## 8. Versioning Policy

* **Patch** (`x.y.Z`) — non-content fixes (formatting, accessibility, padding). Outputs may differ byte-for-byte but are semantically identical.
* **Minor** (`x.Y.0`) — additive columns/sections; older consumers continue to work because parsers tolerate trailing columns.
* **Major** (`X.0.0`) — column rename, semantic change, ATO line-ref change. Old version remains supported in `state = 'published'` for at least 12 months; both are listed in the registry.

Old versions are **never** mutated. A report rendered under v1.2.0 will always be reproducible under v1.2.0 even if v2.0.0 is current, provided the engine and ruleset versions are still available (they always are — engine versions are immutable on npm, rulesets are immutable in the DB).

## 9. Renderer API

```ts
// apps/web/server/reports/render.ts
import type { ScenarioResultId, TemplateSlug, TemplateVersion, Locale } from '@/types';

export interface RenderRequest {
  scenarioResultId: ScenarioResultId;
  templateSlug:     TemplateSlug;
  templateVersion:  TemplateVersion;
  locale:           Locale;          // 'en-AU'
  requestedBy:      UserId;
  tenantId:         TenantId;
  purpose:          'personal' | 'adviser' | 'data_export';
}

export interface RenderResult {
  reportId:        string;
  format:          'pdf' | 'csv' | 'json' | 'xlsx';
  outputHash:      string;  // sha256 over canonical content
  storageKey:      string;  // Supabase Storage object key
  byteLength:      number;
  presignedUrl:    string;  // 7-day expiry
  templateVersion: TemplateVersion;
  rulesetVersion:  string;
  engineVersion:   string;
  generatedAt:     string;  // ISO; excluded from outputHash
}

export async function render(req: RenderRequest): Promise<RenderResult>;
```

The renderer:

1. Loads the scenario result (RLS-enforced); refuses if the requester lacks read access.
2. Loads the template row; refuses if `state != 'published'`.
3. Loads the disclaimer text by `disclaimer_ref`; refuses if missing.
4. Renders the artifact to a buffer.
5. Computes the output hash (content + disclaimer + identification, excluding `generated_at`).
6. Writes to Supabase Storage under `tenants/{tenant_id}/reports/{report_id}.{ext}`, ACL `private`.
7. Inserts a row in `reports` and `audit_logs` with full provenance.
8. Returns a presigned URL.

## 10. Storage & Retention

| Class                       | Bucket / Prefix                                | Retention                                | Encryption        |
| --------------------------- | ---------------------------------------------- | ---------------------------------------- | ----------------- |
| Standard reports            | `tenants/{tenant_id}/reports/`                 | 24 months from generation                | AES-256 at rest   |
| Data export (APP 12)        | `tenants/{tenant_id}/data-exports/`            | 14 days (auto-purged)                    | AES-256 + per-tenant KMS key |
| Audit-pinned reports        | `tenants/{tenant_id}/audit/`                   | 7 years (cold storage after 1 year)      | AES-256 + KMS     |
| Synthetic / test outputs    | `synthetics/`                                  | 7 days                                   | AES-256           |

Presigned URLs default to 7-day expiry; data-export URLs to 24 hours. URL minting is rate-limited per user per day to prevent enumeration.

## 11. Accessibility

* PDFs are tagged (PDF/UA where the renderer supports it); headings have semantic levels; tables have header rows declared; alt text on charts cites the companion table.
* CSV/XLSX are inherently accessible; the metadata header lets screen readers announce provenance.
* Colour usage matches `/ui-ux/data-viz-guidelines.md` — never colour-only encoding; companion tables accompany all charts in PDFs.

---

## 12. Anti-Patterns (Renderer Refuses)

* Emitting any output without a disclaimer block.
* Using `Date.now()` inside content (only the footer is allowed a generated-at).
* Including raw email, TFN, full DOB, or card data.
* Using fonts that are not embedded.
* Embedding remote images (renderer fetches and inlines, then strips remote refs).
* JavaScript inside PDFs (action handlers, embedded scripts) — flat refusal.
* Outputs whose `output_hash` differs between two consecutive renders of identical inputs (CI golden test catches this).
