=== Fails-closed proof ===

assertValidRawRuleset() in adapter.ts is a TypeScript `asserts r is T`
runtime assertion function. It is NOT a type cast. The `as Record<string,
unknown>` on line 57 is a TypeScript narrowing to enable property access;
every field check that follows is a JavaScript typeof/Array.isArray/regex
runtime test that throws a TypeError on failure.

RS-07 and RS-13 demonstrate this by bypassing TypeScript via:
  `as unknown as RawRuleset`  (RS-07)
  `JSON.parse(...) as unknown as RawRuleset`  (RS-13)
In both cases the throws still fire — proving no TypeScript erasure path
lets malformed data through silently.

=== Deviations ===

DEV (to be formally logged in deviation-log.md at D05-T4):
  fy2026.schema.json exists as documentation of the ruleset structure but
  is NOT used for runtime JSON Schema validation. Runtime validation is
  performed by assertValidRawRuleset() in adapter.ts (TypeScript `asserts`
  function with typeof/regex checks). No external schema-validation library
  (e.g. ajv) is used; this keeps the engine package dependency-free.
  The JSON Schema file may drift from the runtime assertions — it is
  documentation only and should not be treated as authoritative.
  Disposition: accepted. Schema can be wired if an audit requires it.
